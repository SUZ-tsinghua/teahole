壹 取舍 The trade-off

两条要求互相拉扯：严格准入，与默认不可链。 常见做法「登录后用昵称发言」过不了第二关—— posts.user_id = 42 一行就把人卖了。

壹把「证明你是成员」和「证明你能写入」拆开。

登录后你拿到一枚长期 cookie，只用来证明你是茶园成员； 写帖子、评论、共享文档或反应前另外领一枚随机 32 字节令牌，服务器只存它的 SHA-256。 凭据是分离的，账号永远不会出现在普通成员的帖子和评论旁边。

posts.user_id = 42 ← 行不通 posts.pseudonym = "a3f9b2c1" ← teahole

每 10 分钟一次清扫，过期令牌哈希被删除，FK 级联把反应、评论反应和投票明细带走。 帖子 / 评论的编辑指纹与文档的创建者删除凭据也会被清空—— 那一刻起，旧内容在数据库里再也找不到对应的登录账号。

02How it runs · 流程

从注册到发帖和文档，
一条窄路。

每一步都只交付当下需要的东西，不留多余的关联。普通成员的内容写入都要带令牌，服务器永远当场把它哈希掉。

壹 · 注册

邮箱白名单 · 验证码登录

邮箱 SHA-256 必须命中内存里的白名单；验证码先通过邮箱确认，成功注册后邮箱作为登录名写入账号表。设置里可以改邮箱：旧邮箱进「曾用邮箱」列表，登录时只要当前邮箱或任意一个曾用邮箱仍在白名单里就放行。

贰 · 领取

申领一枚写入令牌

点「领取/轮换」拿到 32 字节随机串。服务器只存哈希 + 过期时间，外加你账号上的当日计数——明文不再回到任何地方。

叁 · 发言

带令牌写帖、评论、文档、反应

前 8 位哈希成为你这一天的匿名 ID。帖子是 #post123，共享文档是 #doc42；管理员发言会署名公开。

肆 · 失效

24h 后，连同关联一起拆掉

过期那一刻，帖子 / 评论的编辑删除权限随令牌一起消失，反应和投票明细被级联拆除；共享文档保留内容，但创建者删除凭据会归零。

i.

时间相关性。

能拿到 web 日志或网络抓包的运维，可以把「某用户取了令牌」与「某令牌发了帖」对上。真正的不可链需要盲签名方案——teahole 只让数据库不可链。

ii.

管理员发言公开。

这是产品取舍而非缺陷——@admin 直接署名，方便反馈。匿名规则只对普通成员生效。

iii.

共享文档是协作面。

文档会显示公开的创建者 / 最后编辑者 pseudonym，适合沉淀资料，不等价于匿名帖子。

iv.

令牌存在 localStorage。

清浏览器 = 当日剩余配额一起没。要重新申领，会再吃掉一枚每日上限（默认 5 枚）。

v.

滥用上限。

每日令牌帽位限制了「同一用户内的 sybils」。管理员能删内容，但仍然不能反查到具体账号。
