# 茶园树洞 / teahole

> [English](README.en.md) · 中文

> **仅限内部访问。** 为了仅让茶园内部访问，只有 allowlist 里的邮箱才能注册。
> 想体验的朋友，请发邮件或微信找我

一个「登录可见、默认匿名」的内部社区。在线访问：<https://teahole.fly.dev>

最重要的两个入口是：

- **匿名发帖** — 用 24 小时令牌写帖子、评论、反应和投票，普通成员的内容不回链账号。
- **共享文档** — 用同一套 Markdown / LaTeX / 图片能力维护 wiki 式资料，任何有效令牌可编辑，账号只承担准入和阅读偏好。

普通用户通过短期令牌发帖、评论并参与共享文档；账号不会和普通成员的帖子 / 评论形成数据库关联路径；
管理员可以选择用用户名直接发言，方便大家 `@` 反馈。

## 为什么要这样做

两条硬需求天然矛盾：

- **严格访问控制** — 只有登录用户能读、能发帖、能编辑共享文档。
- **默认不可追溯** — 普通成员的帖子和评论，不应该能在数据库里连回某个账号。

「先登录、再用昵称发帖」的常见做法过不了第二条：数据库里一行 `posts.user_id = 42`
就把一切暴露了。本项目的做法是把**认证**和**写入授权**解耦 —— 用短期的、
和账号无关联的令牌来授权发帖、评论、文档编辑等操作。

## 你看到的网页是怎么跑的

1. **注册 / 登录**
   - 用邮箱注册，邮箱必须在服务端的 allowlist 里（allowlist 文件里的邮箱
     只以 sha256 存在内存，不写库也不打 log）。
   - 注册时会发 6 位验证码到邮箱（通过 Gmail SMTP），验证通过才能设密码。
   - 登录后得到一枚长期 session cookie，只用来证明「你是合法成员」。
2. **领取写入令牌**
   - 进来之后先点左下角「获取 / 轮换令牌」，拿到一枚随机 32 字节的令牌。
   - 服务器只存 `sha256(token)` 和过期时间，以及你今天**领了几次**的计数 —— 不存原始令牌，也不存「这枚令牌属于哪个账号」的映射。
   - 原始令牌只返回你一次，客户端放在 `localStorage`。
3. **发帖 / 评论 / 共享文档 / 反应 / 投票**
   - 每次写操作都带上令牌。服务器按 hash 查令牌、判断有效，然后写入：
     - 普通成员：`pseudonym = sha256(token)[:8]`，一个 8 位十六进制的匿名 ID。
     - 管理员：`pseudonym = username`，直接用用户名署名。
   - 任何时候数据库里都没有 `user_id` 和帖子 / 评论并列。共享文档会记录公开的创建者 / 最后编辑者 pseudonym，但也不存用户 ID。
4. **令牌过期**
   - 每 10 分钟扫一遍，删除过期的令牌哈希（默认 TTL 24 小时）。
   - 令牌过期那一刻，「这条帖子 / 评论是哪枚令牌写的」就再也查不到了 —— 同时你也
     不能再编辑/删除用那枚令牌发的内容。
   - 共享文档不会随令牌过期消失；只是创建者令牌哈希会被清掉，之后只有管理员能删除，任何有效令牌仍可继续编辑。

## 功能

### 内容

- **匿名发帖** — 帖子、嵌套评论、反应和投票都走短期令牌；普通成员只留下公开 pseudonym，不留下账号外键。
- **共享文档** — `/d/<id>` 是 wiki 式资料页，支持频道归类、收藏、全文检索；任何持有效令牌的成员都能编辑，创建者令牌仍有效时或管理员可删除。
- **Markdown 渲染** — 粗体、斜体、行内 / 围栏代码、链接、列表、引用、h1–h3。
  原始 markdown 保存在库里，HTML 只在客户端生成。
- **代码高亮** — 围栏代码块自动着色。支持 js/ts、Python、C 系（C/C++/Rust/Go/Java/Swift/Kotlin）、SQL、Shell、JSON、CSS；
  未识别语言显示为纯文本不报错。
- **LaTeX 公式** — 行内 `$E=mc^2$`，块级用两行 `$$` 围起来，由 KaTeX 渲染（同源加载，不走 CDN）。
- **图片上传** — 上传时剥离 EXIF 等元信息，按内容 hash 存盘，不按用户归档。
- **搜索** — 帖子标题 / 正文 / 评论和共享文档标题 / 正文都可搜。较长查询走 FTS5 trigram 排序，短查询自动退回子串扫描；没有最小字数限制，搜索词本身不进日志。
- **ID 引用** — 帖子显示为 `#post123`，共享文档显示为 `#doc42`；正文里写这些 ID 会变成可点击 chip。旧式 `#123` 仍按帖子处理，搜索框里输入 `#123` 也会直达帖子。

### 交互

- **嵌套评论** — 评论可以回复评论。
- **`@` / `#` 提醒** — 在输入框里支持 `@abc12345`（匿名 ID）/ `@admin`（管理员用户名）/ `#post123` 自动补全；正文中的 `#doc42` 也会渲染成文档链接。
- **收件箱** — 发往你当前匿名身份（或管理员用户名）的 `@提醒` 会进「提醒」栏，24 小时内可见。
- **反应** — 👍 / 👎 两种固定反应，帖子和评论都可用；一枚令牌对同一目标、同一种反应只能投一票。
  令牌过期时通过外键 cascade 自动清除。「我投过没」只在本地 `localStorage` 里，服务器的读接口不会返回任何用户粒度的状态。
- **投票** — 发帖时可附 2–10 个选项。每枚令牌每个投票只能投一次；明细票随令牌过期清掉，累计票数留在选项上。
- **编辑 / 删除自己的内容** — 用令牌做权限校验，只要令牌还没过期就能改、过期了就改不动。

### 组织

- **频道** — 管理员建立的话题分区。每个用户可以**置顶**常看的频道、**静音**不关心的频道，还能看到**未读计数**。
- **收藏帖子 / 文档** — ★ 收藏到账号上，换令牌不会丢。
- **公告栏** — 管理员专用的站内公告，侧栏常驻。
- **RSS** — `/api/feed.xml` 给登录用户提供最近 50 条帖子；不绑定请求者身份，也没有分页游标。
- **深链接** — 帖子 `/p/<id>`、公告 `/b/<id>`、共享文档 `/d/<id>` 都由前端路由；后端 SPA catch-all 无论 id 是否存在都返回 `index.html`（不泄露存在性）。

### 端

- **桌面 + 移动端网页** — 同一份 SPA，移动端布局单独适配。
- **深色 / 浅色主题** — 右上角切换；跟随系统偏好。

## 数据库里任一时刻有什么

| 表             | 能追回账号？                   | 存了什么                                  |
|----------------|-------------------------------|-------------------------------------------|
| `users`        | 是                            | 邮箱、bcrypt 密码、每日令牌计数、密码版本     |
| `post_tokens`  | 否                            | `sha256(token)`、`expires_at`             |
| `posts`        | 普通成员：否 / 管理员：是     | `pseudonym`、标题、正文、频道、时间、临时 `token_hash` |
| `comments`     | 普通成员：否 / 管理员：是     | `pseudonym`、正文、`post_id`、`parent_id`、临时 `token_hash` |
| `posts_fts` / `comments_fts` / `docs_fts` | 否（派生索引） | 标题 / 正文 / 评论的全文检索索引             |
| `reactions`    | 否（以 `token_hash` 外键 cascade） | 帖子 / 令牌 / 反应种类                |
| `comment_reactions` | 否（以 `token_hash` 外键 cascade） | 评论 / 令牌 / 反应种类            |
| `polls` / `poll_options` | 否                    | 所属帖子、题目、选项、累计票数              |
| `poll_votes`   | 否（以 `token_hash` 外键 cascade） | 投票 / 令牌 / 所选选项              |
| `mentions`     | 否（目标是 pseudonym）         | 目标匿名 ID、来源帖子 / 评论、时间          |
| `docs`         | 否（仅公开 pseudonym / 临时令牌哈希） | 标题、正文、频道、创建者 / 编辑者 pseudonym |
| `saved_posts`  | 是（按账号收藏）               | 用户、帖子                                |
| `saved_docs`   | 是（按账号收藏）               | 用户、共享文档                             |
| `channels`     | 否                            | 频道 slug、名称、描述                      |
| `user_channel_prefs`| 是（按账号 pin / mute / 已读） | 用户、频道、偏好、读取水位             |
| `bulletins`    | 是（管理员署名）               | 管理员发布的站内公告                        |
| `email_codes`  | 是（短期注册流程）             | 邮箱、验证码哈希、过期时间、尝试次数          |
| `revoked_sessions` | 否（短期会话清理）          | 已退出 session 的 `jti`、过期时间           |

普通成员的 `posts` / `comments` 没有通向 `users` 的连接路径。共享文档是协作资料面，
保留的是公开 pseudonym 和临时删除权限哈希，不保留用户 ID。阅读类偏好
（收藏 / 频道 pin / mute / 已读水位）故意用账号维度，因为这些是读者端状态，
**不涉及作者身份**。

## 已知局限

- **时序关联**。能看到 web server access log / 网络抓包的攻击者，可以把
  「用户 X 在 14:02:11 请求 `/api/token`」和「14:02:14 有人用某枚令牌发
  帖」关联起来。真正意义上的不可追溯需要盲签之类的协议；本项目只保证
  **数据库**里的内容不可追溯。
- **管理员发言是公开的**。这是产品上有意为之，不是漏洞。
- **共享文档是协作面**。文档会显示创建者 / 最后编辑者的公开 pseudonym；它适合沉淀资料，不等价于匿名帖子。
- **令牌丢失**。清浏览器 `localStorage` 就会丢，今天剩余的发帖配额也
  跟着失效，只能重新领（计入每日上限）。
- **滥用**。每日令牌上限（默认 5 枚）限制了一个账号能创建的匿名身份数量。
  管理员只能删内容，不能把匿名发言反查成具体用户。

## 本地运行

```bash
npm install
JWT_SECRET=$(openssl rand -hex 32) npm start
# 打开 http://localhost:3000
```

如果只是本地临时体验，可以显式打开调试账号种子：
`DEV_SEED_USERS=1 JWT_SECRET=$(openssl rand -hex 32) npm start`。这会重置
`users` 表并创建 `admin@example.com / admin1234` 和
`user@example.com / user1234`，**生产环境不要设置这个变量**。

环境变量：

| 变量                  | 默认         | 作用                                             |
|----------------------|--------------|--------------------------------------------------|
| `PORT`               | `3000`       | 监听端口                                         |
| `JWT_SECRET`         | 启动时随机    | 生产环境务必自己设；用默认会让重启后所有 session 失效 |
| `TOKEN_TTL_HOURS`    | `24`         | 写入 / 发帖令牌寿命                                |
| `MAX_TOKENS_PER_DAY` | `5`          | 每个账号每天最多领几枚令牌                         |
| `GMAIL_USER`         | —            | 发验证码用的 Gmail 账号                            |
| `GMAIL_APP_PASSWORD` | —            | Google 的应用专用密码（不是登录密码）               |
| `GMAIL_FROM`         | —            | 覆盖发件人显示名                                   |
| `DEPT_ALLOWLIST_FILE`| `./dept-allowlist.txt` | 注册白名单文件路径                    |
| `TRUST_PROXY`        | —            | 反向代理后填 `1` 或 CIDR，让 `req.ip` 取真实客户端 IP |
| `DEV_SEED_USERS`     | —            | 仅本地调试：设为 `1` 时重置并种下调试账号；生产勿设       |

未设 `GMAIL_USER` / `GMAIL_APP_PASSWORD` 时，验证码直接打到服务端日志，方便本地开发。

### 首个管理员

```bash
sqlite3 data.db "UPDATE users SET is_admin = 1 WHERE username = 'you';"
```

### 注册白名单

把允许注册的邮箱一行一个写进 `dept-allowlist.txt`（`#` 开头是注释）。
文件不存在 = 注册全关闭。服务端每 2 秒 `stat` 一次，可以热改。
如果手里是 CSV / TXT / HTML 名册导出，可以用
`npm run allowlist:import -- roster.csv` 抽取邮箱；加
`--domain mails.tsinghua.edu.cn` 可以只保留指定域名。

## 部署到 fly.io

`Dockerfile` 和 `fly.toml` 已经在仓库里；现网部署在 <https://teahole.fly.dev>。
首次部署：

```bash
fly launch --no-deploy
fly volumes create teahole_data --region sin --size 1
fly secrets set \
  JWT_SECRET=$(openssl rand -hex 32) \
  GMAIL_USER=you@gmail.com \
  GMAIL_APP_PASSWORD='xxxx xxxx xxxx xxxx'
fly ssh console -C 'sh -c "cat > /data/dept-allowlist.txt"' < dept-allowlist.txt
fly deploy
```

后续：`git commit … && fly deploy`。

SQLite 文件落在挂载的 volume 上，机器数必须保持 `min = max = 1`，否则
多实例会分叉数据库。
